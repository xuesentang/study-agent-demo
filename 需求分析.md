大学生概率论与数理统计备考辅导 Agent MVP 需求文档（终版）
一. 产品核心定位 (MVP 版)
一款面向国内本科大一 / 大二学生，基于概率论与数理统计教材与真题，主打 simple 应试模式的交互式学练一体 AI Agent。核心解决大学生概率论与数理统计期末备考时间紧、传统自学效率低、学练分离、知识点割裂、针对性不足的痛点，以「逻辑串联知识点极简铺陈 + 高频考点讲练结合」为核心模式，兼顾应试效率与知识体系完整性，核心差异为全程引导式交互、学练一体、支持用户自定义本校真题题库。

二. 精准目标用户画像 (MVP 版)
•	核心用户：国内本科院校有概率论与数理统计期末备考需求的大一、大二学生
•	用户核心特征：
1.	备考周期短 (集中在考前 1-2 周)，核心目标是高效提分、顺利通过考试，不追求学术深度，极致看重时间性价比
2.	传统看书、刷课、刷题的自学方式专注力低、效率差，容易出现 “学了就忘、换题就懵” 的问题
3.	抓不住本校考试重点，找不到适配的历年真题，无法针对性备考
4.	对 AI 工具接受度高，愿意尝试交互式学习方式，排斥单向灌输式的长内容讲解
•	核心诉求：用最少的时间，完成概率论与数理统计期末考点的系统突击，既掌握高频考点的解题能力，又能形成完整的知识逻辑体系，同时实现学练一体，无需额外花时间刷题。

三. 核心痛点与需求清单 (MVP 版)
3.1 核心痛点
本科大学生概率论与数理统计期末备考时间紧、传统自学效率低、学练分离、知识点逻辑割裂、找不到针对性真题，无法在短时间内完成 “系统知识搭建 + 高频考点突破 + 针对性提分” 的备考全流程。
3.2 对应核心用户需求
1.	能快速获取适配期末考的高频考点精讲，同时通过极简的前置知识点铺垫，形成完整的知识逻辑体系，避免知识点割裂，无冗余内容，最大化节省备考时间
2.	能边学边练，学完核心考点立刻做对应真题测试，获得即时反馈，学练一体，不用额外花时间单独刷题
3.	能有贴合教材逻辑的标准化学习路线，无需自己规划，跟着流程走就能完成全考点的系统突击


四. 核心竞品差异化分析 (MVP 版)
4.1 核心竞品：秘塔 AI「今天学点啥」
竞品核心优势：课程覆盖范围广，参考资料丰富，讲解风格可选择
竞品核心短板 (MVP 精准击穿)：
单向信息输入，无交互式学练环节，用户被动听课，专注力低，学习效率差
学练分离，只有知识点讲解，无配套真题测试与即时反馈，用户学完仍需额外花时间刷题
内容要么全量无差别讲解 (耗时久，不符合突击需求)，要么纯考点堆砌 (知识点割裂，用户无法形成系统认知)
MVP 核心差异化优势
全程交互式学练闭环：边学边测，不完成核心考点测试无法进入下一环节，强制提升专注度与学习效率
体系与效率双兼顾：simple 模式采用「逻辑串联知识点极简铺陈 + 高频考点讲练结合」模式，既不割裂知识体系，又最大化节省备考时间
学练一体无额外负担：考点讲解与真题测试深度绑定，用户完成学习流程即可完成考点掌握与刷题训练，无需额外花时间刷题。

五. 核心功能清单 (MVP 版)
5.1 MVP 必做核心功能 (4 个，无额外冗余功能，可直接落地)
（1）标签化 RAG 知识库模块
1、基于概率论与数理统计教材 + 配套真题搭建，严格按照教材逻辑顺序搭建知识框架；
2. 知识点做双层标签化分类：「逻辑串联型基础知识点 (低频 / 不考)」「核心高频考点」，做内容隔离；
3. 预制贴合教材逻辑的标准化学习路线，明确知识点的先后衔接顺序与高频考点优先级。
（2）交互式学练核心模块 (simple 模式专属)
1. 严格遵循「前置串联知识点极简介绍 (只讲不测)→核心高频考点精讲→对应真题测试→答对进阶 / 答错补讲复测」的固定流程；
2. 串联知识点仅讲解 “核心定义、与下一考点的逻辑关联”，单条内容字数≤100 字，无测试环节；
3. 高频考点精讲聚焦 “核心定义、解题方法、”，单条内容字数≤200 字，配套对应真题测试；
4. 所有回复严格遵循简洁无冗余要求，不引申无关内容.
（3）极简对话交互界面
1. 以对话框为唯一主体，全程由产品主动引导学习流程，无需用户主动规划学习内容；
2. 支持用户通过文字、图片两种形式提交答案；
3. 适配电脑端、手机端浏览器，界面无冗余功能入口
5.2 MVP 绝对不做、后续迭代功能清单
1.	语音输入 / 输出功能
2.	deep 深度学习模式
3.	多教材 / 多学科 / 多课程适配
4.	通用提示词模板功能
5.	可视化知识导图、错题本功能
6.	社区、分享、打卡等社交类功能
7.	复杂的用户等级、积分体系

六. 核心交互流程 (MVP 终版，完整闭环，无断点)
1.	用户进入产品，默认进入「高等数学 simple 应试备考模式」
2.	产品展示贴合教材逻辑的标准化学习路线、全流程预计学习时长，同时提供「上传本校真题定制专属路线」的可选入口，用户确认后直接进入学习流程
3.	产品推送当前考点的前置逻辑串联知识点极简介绍，仅讲解核心定义与后续考点的逻辑关联，无测试环节，用户确认知晓后即可进入下一步
4.	产品推送对应核心高频考点的极简精讲，聚焦考频、核心定义与解题方法，无冗余内容
5.	精讲完成后，产品立刻推送该考点的配套真题测试，用户通过文字 / 图片形式提交答案，产品实时反馈是否正确，若错误则补讲复测并测试一道新题，
6.	产品即时批改答案，判断用户考点掌握情况：
•	答对：直接进入下一知识点的学习流程，循环步骤 3-5
•	答错：推送极简错题讲解与解题思路，再推送 1 道同考点复测题，用户答对后进入下一知识点学习
7.	测试完成后，给用户输出极简的考点掌握情况报告，核心备考闭环完成

七. 非功能需求 (MVP 版，仅保留落地必备底线要求)
1.	内容准确性要求：用户学习某一章节 / 考点时，不得出现其他章节 / 科目的内容；考点讲解、题目答案与批改的准确率 ≥ 95%
2.	知识连贯性要求：全学习路线严格遵循教材的逻辑递进顺序，所有核心高频考点必须有对应的前置串联知识点铺垫，无逻辑断层、无知识点割裂
3.	响应速度要求：单条消息回复时长≤3 秒，用户上传试卷后的识别处理时长≤1 分钟
4.	数据安全要求：用户上传的试卷、学习记录、答题内容仅对用户本人可见，不对外泄露，不用于非产品功能的其他用途
5.	内容简洁性要求：逻辑串联知识点单条讲解字数≤100 字，高频考点精讲单条字数≤200 字，无长篇大论、无无关引申内容

八. 需求验收标准 (MVP 版，全可量化、可验证，无模糊描述)
1.	核心闭环验收：用户可完整走完「进入产品→学习全流程→最终验收测试」的完整闭环，无流程断点；全程由产品主动引导，无需用户主动规划学习内容与学习顺序
2.	功能验收：3 个核心功能全部可正常使用，无严重 bug； 
3.	内容与逻辑验收：
•	全学习路线严格遵循同济高数教材的逻辑顺序，无逻辑断层；所有核心高频考点均有前置串联知识点铺垫，铺垫内容仅讲解、不设置测试
•	知识点讲解与题目答案准确率≥95%，无跨章节、跨科目串题情况
•	严格遵守内容简洁性要求，单条讲解内容不超字数上限，无冗余无关内容
4.	用户体验验收：单个知识点的完整学习闭环 (含铺垫 + 精讲 + 测试) 时长≤5 分钟，完整的概率论与数理统计备考全流程时长≤8 小时，符合期末突击的时间要求

九. 迭代规划预留 (双轨增值部分，不占用 MVP 开发资源，可用于简历 / 赛事答辩)
•	V1.1 版本迭代：新增 deep 深度学习模式，适配考研 / 保研学生的深度理解需求；拓展线性代数、概率论与数理统计等其他大学公共数学课程
•	V1.2 版本迭代：新增可视化知识导图、专属错题本、学习数据复盘功能，优化用户学习沉淀能力；新增语音输入 / 输出功能，优化多场景使用体验
•	V2.0 版本迭代：拓展 AI 前沿知识、职业技能等学习内容，覆盖大学生全周期学习需求，从单科应试辅导 Agent 升级为大学生全周期学习辅导 Agent
